# 多语言贡献方法

## 项目国际化架构

WinConfigBKP 使用 Qt 的 `QTranslator` 框架实现多语言支持。

- **翻译数据**: Qt `.ts` (XML) + `.qm` (编译后二进制) 文件
- **运行时**: 通过 `src/utils/i18n.py` 模块加载 `.qm` 文件
- **语言切换**: 设置页面 ComboBox → 切换 locale → 热加载翻译 → 所有界面即时刷新
- **翻译函数**: `tr("中文文本")` — 取代所有硬编码的 UI 字符串

### 文件位置

```
lang/
├── zh_CN.ts          # 中文源翻译文件（空翻译 = 显示源代码文本）
├── en_US.ts          # 英文翻译文件（含实际翻译）
├── en_US.qm          # 编译后的英文翻译
├── extract_strings.py  # 从源代码提取 tr() 字符串，生成 .ts 文件
├── fill_translations.py # 将翻译词典写入 en_US.ts，编译 .qm
└── release.bat       # 一键编译所有 .ts → .qm
```

## 如何添加新语言的翻译

### 第 1 步：创建 .ts 文件

先运行提取脚本，生成包含所有翻译条目的 `.ts` 文件：

```powershell
python lang/extract_strings.py
```

这会重新生成 `zh_CN.ts` 和 `en_US.ts`。`zh_CN.ts` 是源语言（中文），所有 `<translation>` 标签留空（Qt 遇到空翻译时自动显示源代码文本）。

### 第 2 步：添加语言选项

在 `src/gui/settings_tab.py` 的语言 ComboBox 中添加新语言：

```python
self.lang_combo.addItem("Français", "fr_FR")
self.lang_combo.addItem("日本語", "ja_JP")
```

第一个参数是显示名称，第二个是 locale 代码（与 `.ts/.qm` 文件名一致）。

### 第 3 步：创建翻译文件

复制 `en_US.ts` 为 `lang/fr_FR.ts`（或其他 locale 代码），将 `<translation>` 中的英文替换为目标语言。

或手动创建：

```xml
<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE TS>
<TS version="2.1" language="fr_FR">
<context>
    <name>App</name>
    <message>
        <source>备份</source>
        <translation>Sauvegarder</translation>
    </message>
    <!-- 其他条目... -->
</context>
</TS>
```

### 第 4 步：编辑翻译词典（可选）

`lang/fill_translations.py` 中有一个 `TRANSLATIONS` 字典，用于一键填充 `en_US.ts`。如果要为其他语言创建类似的填充脚本，可以复制此文件并修改字典内容。

### 第 5 步：编译为 .qm

```powershell
# 方式一：使用 release.bat
lang\release.bat

# 方式二：手动编译
pyside6-lrelease lang/fr_FR.ts -qm lang/fr_FR.qm
```

编译后将生成 `.qm` 二进制文件，这是运行时实际加载的格式。

### 第 6 步：更新加载逻辑

在 `src/utils/i18n.py` 中，`install()` 函数会自动尝试加载 `lang/<locale>.qm`：

```python
qm_path = Path(__file__).resolve().parent.parent.parent / "lang" / f"{locale}.qm"
if qm_path.exists():
    new_translator = QTranslator()
    if new_translator.load(str(qm_path)):
        _translator = new_translator
        app.installTranslator(_translator)
```

`zh_CN` 是源语言，不会加载 `.qm` 文件。其他语言会自动加载对应的 `.qm`。

## 开发指南

### 在代码中使用 tr()

```python
# 导入
from src.utils.i18n import tr, on_locale_changed, off_locale_changed

# 在函数/方法中使用
self.status_label.setText(tr("就绪"))
QMessageBox.warning(self, tr("提示"), tr("请先选择要备份的配置"))
self.signals.message.emit(tr("正在计算文件哈希..."))
```

**注意事项**:

- 所有**用户可见**的字符串都必须用 `tr()` 包裹
- `logger.debug()` 日志不应使用 `tr()`（日志面向开发者）
- 异常消息可以保留中文，不必翻译（开发者面向的异常显示在日志中）
- `f-string` 与 `tr()` 不兼容：用 `tr("模板 {}").format(值)` 替代 `f"...{值}..."`
- 字符串模板放在 `tr()` 内部，参数用 `.format()` 传入

```python
# 正确
self.label.setText(tr("发现 {} 个文件").format(count))

# 错误 - 不要这样做
self.label.setText(f"发现 {count} 个文件")  # ❌ 不会翻译
self.label.setText(tr(f"发现 {count} 个文件"))  # ❌ f-string 在 tr() 内
```

### 添加对语言切换的支持

需要即时刷新的界面组件，按以下模式添加 i18n 生命周期：

```python
class MyWidget(QWidget):
    def __init__(self, ...):
        super().__init__()
        self._setup_ui()
        self.retranslate_ui()                          # 1. 初始翻译
        self._retranslate_cb = self.retranslate_ui     # 2. 保存回调引用
        on_locale_changed(self._retranslate_cb)         # 3. 注册回调
        self.destroyed.connect(self._on_destroy)        # 4. 清理

    def retranslate_ui(self):
        """设置/更新所有用户可见文本（在构造和语言切换时调用）。"""
        self.some_button.setText(tr("备份"))
        self.some_label.setText(tr("状态: 就绪"))
        self.group_box.setTitle(tr("设置"))
        # 更新 ComboBox 时注意保留当前选择索引
        idx = self.combo.currentIndex()
        self.combo.clear()
        self.combo.addItems([tr("选项A"), tr("选项B")])
        if 0 <= idx < self.combo.count():
            self.combo.setCurrentIndex(idx)

    def _on_destroy(self):
        off_locale_changed(self._retranslate_cb)  # 必须清理回调避免悬空引用
```

### 热切换机制

`src/utils/i18n.py` 提供以下 API：

| 函数 | 说明 |
|------|------|
| `tr(key)` | 翻译函数，返回当前语言的翻译文本 |
| `install(locale)` | 安装指定 locale 的翻译器，触发所有回调 |
| `current_locale()` | 返回当前 locale 代码 |
| `on_locale_changed(cb)` | 注册回调，语言切换时自动调用 |
| `off_locale_changed(cb)` | 取消注册回调 |

## 更新翻译流程

当代码中的 `tr()` 调用发生变化时（新增/修改/删除字符串）：

```powershell
# 1. 重新提取所有 tr() 字符串
python lang/extract_strings.py

# 2. 用翻译词典填充英文翻译
python lang/fill_translations.py

# 3. 查看未翻译的条目
#    (运行 fill_translations.py 时会打印未翻译列表)

# 4. 编译 .qm 文件
lang\release.bat

# 5. 提交所有变更
git add lang/ src/utils/i18n.py
git commit -m "feat(i18n): 更新翻译文件"
```

### 为 fill_translations.py 添加新翻译

`lang/fill_translations.py` 中的 `TRANSLATIONS` 字典维护中文→英文翻译映射。当添加新字符串后，运行提取脚本会发现未翻译条目。在字典中添加对应的翻译：

```python
TRANSLATIONS: dict[str, str] = {
    # ...
    "新增的中文字符串": "New English string",
}
```

然后重新运行 `fill_translations.py` 即可填充到 `en_US.ts`。

## 边界情况

### 列表项/复选框文本

列表项是动态的（从 JSONC 配置读取的名称），不需要翻译。但**列表的标题**（如表头、GroupBox 标题）需要翻译。

### ComboBox 选项

ComboBox 选项在 `retranslate_ui` 中重置时，需要使用 `currentIndex()` 保存并恢复选中状态，避免语言切换时选中项跳变。

### 格式化字符串

对于包含占位符的字符串（如 `"{} 个文件"`），保持 `tr()` 包裹整个模板，不要将格式化部分拆出来。

### 空翻译 = 源代码文本

Qt 的 `QTranslator` 行为：如果 `.ts` 文件中某条目的 `<translation>` 为空，运行时将显示 `<source>` 文本。因此 `zh_CN.ts` 可以全部留空——中文就是源代码语言。

### 语言标签显示

语言 ComboBox 中的语言名称（如 "中文"、"English"）在切换语言后仍会保持原样，这是有意的——用户需要能识别当前语言选项。如果在英文模式下将 "中文" 改为 "Chinese"，当用户切换到中文时，又需要显示 "中文"，这会创建循环依赖。
