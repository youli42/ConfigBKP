# 08 — 调试模式使用指南

## 启用方式

程序通过 Python 标准库 `logging` 输出运行日志，通过环境变量 `WINCONFIGBKP_DEBUG` 控制日志级别。

```powershell
# 源码运行
$env:WINCONFIGBKP_DEBUG = "1"
python main.py

# 或一行命令
$env:WINCONFIGBKP_DEBUG = "1"; python main.py

# 打包 exe 运行
set WINCONFIGBKP_DEBUG=1
WinConfigBKP-1.0.0.exe
```

## 日志级别

| 环境变量 | 日志级别 | 输出内容 |
|----------|----------|----------|
| 未设置（默认） | `INFO` | 仅程序启动等基本信息，无调试输出 |
| `WINCONFIGBKP_DEBUG=1` | `DEBUG` | 输出所有调试日志到 `stderr` |

日志格式：`[LEVEL] 消息内容`

## 日志输出位置

日志写入标准错误流（`stderr`），与 GUI 界面互不干扰：

- 终端运行时直接显示在控制台
- 打包后的 exe 在终端启动时显示，双击运行时需重定向查看：
  ```powershell
  WinConfigBKP-1.0.0.exe 2>debug.log
  ```

## 各模块日志点

| 模块 | 日志内容 |
|------|----------|
| `backup_engine.py` | 批处理开始/结束、各配置文件名、文件变化数、版本裁剪、异常信息 |
| `restore_engine.py` | 备份读取文件数、文件占用检测结果、恢复文件数、异常信息 |
| `scanner.py` | 扫描规则数、每条路径的存在检查结果、模糊匹配结果、最终匹配数 |
| `manifest_manager.py` | 每文件的变更状态（新文件/已变更/未变）、增量统计 |
| `file_locker.py` | 文件占用检测、MoveFileExW 调用参数与返回值 |

## 常见调试场景

### 场景 1：扫描后某规则未自动勾选

```powershell
$env:WINCONFIGBKP_DEBUG = "1"; python main.py
# 点击「扫描本机已装软件」
# 看 stderr 输出：  - xxx: C:\path (不存在)
# 说明该配置规则的 paths 在本机找不到对应文件
```

### 场景 2：备份后某配置未出现在历史中

```powershell
$env:WINCONFIGBKP_DEBUG = "1"; python main.py
# 勾选并执行备份
# 看 stderr 输出：  [xxx] 源文件路径不存在，跳过
# 说明该配置的源文件路径在本机不存在，或写入到了不同的存储路径
```

### 场景 3：恢复时文件被锁定

```powershell
$env:WINCONFIGBKP_DEBUG = "1"; python main.py
# 执行恢复操作
# 看 stderr 输出：  [xxx] N 文件被占用: [Code.exe]
# 说明 VS Code 正在运行，关闭后重试
```

## 注意事项

- DEBUG 日志的输出对程序性能影响极小，`logging` 模块在级别未达到时内部短路，不评估格式化参数。
- 所有日志代码都会被打包进 exe，但生产环境下日志级别为 INFO，DEBUG 级代码在运行时不执行。
- 无需为调试模式做任何额外配置或安装。
