# main.py / cli.py

## main.py

文件职责：程序入口。使用 `argparse` 解析 `--silent-backup` 参数；检测管理员权限并提权；启动 PySide6 QApplication 进入 GUI 主循环或调用 CLI 模式。在进入主循环前配置 logging 根日志器。

### 公开接口

| 函数/方法 | 参数 | 返回值 | 核心作用 | 副作用/异常 |
|-----------|------|--------|----------|-------------|
| `is_admin()` | 无 | `bool` | 调用 `shell32.IsUserAnAdmin` 判断当前进程是否有管理员权限 | `AttributeError` 时返回 `True`（非 Windows 平台兼容） |
| `main()` | 无 | `None` | 解析 `--silent-backup`；配置 `logging.basicConfig`；非管理员 → `subprocess.list2cmdline` 提权重启；管理员 → 启动 GUI 或 CLI | `--silent-backup` 时调用 `run_silent_backup()` 后直接 return |

### CLI 模式（--silent-backup）

```powershell
python main.py --silent-backup
```

不启动 GUI 窗口，使用 `QCoreApplication` + `QThreadPool` 后台执行 `BatchBackupWorker`，完成后自动退出。适用于 Windows 计划任务调用。

## cli.py

文件职责：`run_silent_backup()` 函数，与 GUI 无关的所有批处理备份逻辑。从 main.py 分离，避免 main.py 顶部 import 膨胀。

### 公开接口

| 函数 | 参数 | 返回值 | 核心作用 |
|------|------|--------|----------|
| `run_silent_backup()` | 无 | `None` | 加载所有 enabled 配置规则 → 收集文件 → 创建 BatchBackupWorker → QThreadPool 后台执行 → 完成后日志输出统计 |

## logging 配置

```python
logging.basicConfig(
    level=logging.DEBUG if os.environ.get("WINCONFIGBKP_DEBUG") else logging.INFO,
    format="[%(levelname)s] %(message)s",
    stream=sys.stderr,
)
```

设置环境变量 `WINCONFIGBKP_DEBUG=1` 可输出 DEBUG 级别的日志到 stderr。生产环境默认 INFO 级别，DEBUG 日志被运行时跳过。详细使用指南见 `使用文档/08-调试模式使用指南.md`。

## 特别说明

- ⚠️ **管理员权限**：`main()` 中调用 `ctypes.windll.shell32.ShellExecuteW` 提权，参数 `runas` 触发 UAC 对话框。使用 `subprocess.list2cmdline(sys.argv)` 正确转义含空格的参数。
- `--silent-backup` 模式不请求管理员权限（由计划任务运行时通常已有 SYSTEM 权限）。
