# 04-gui — 界面层

## main_window.py

文件职责：组装 3 个 Tab（备份/恢复、规则管理、设置）到 QTabWidget 中。切换 Tab 时触发对应视图的数据刷新。

### 公开接口

| 类/方法 | 参数 | 返回值 | 核心作用 |
|---------|------|--------|----------|
| `MainWindow.__init__` | 无 | — | 创建 `config/` 和 `backups/` 目录，初始化 LocalStorage，组装 3 个 Tab |
| `_on_tab_changed` | `index: int` | `None` | Tab 0 → 刷新配置列表；Tab 1 → 刷新规则列表 |

### 特别说明

- ⚠️ **Tab 切换信号（currentChanged）仅切换视图，不涉及任何备份/恢复业务逻辑。**
- `get_config_dir()` 返回 `config/` 目录（自动创建 `user/` 子目录）。
- `get_default_backup_dir()` 返回 `backups/` 目录。

## home_tab.py

文件职责：主操作面板。左侧展示所有配置规则的勾选列表；右侧显示备注输入、备份目标选择、备份/恢复按钮、进度条；下方为历史记录表格。

### 公开接口

| 类/方法 | 参数 | 返回值 | 核心作用 |
|---------|------|--------|----------|
| `HomeTab.__init__` | `config_dir: Path`, `storage: StorageBackend` | — | 初始化界面，调用 `refresh_configs` 加载规则 |
| `refresh_configs()` | 无 | `None` | 从 `config/builtin/` 和 `config/user/` 加载 `.jsonc` 规则并创建勾选框 |
| `_scan()` | 无 | `None` | 调用 `scan_installed`，自动勾选匹配的规则，更新状态文本 |
| `_select_all()` / `_deselect_all()` | 无 | `None` | 全选/取消全选左侧配置勾选框 |
| `_select_all_detail()` / `_deselect_all_detail()` / `_invert_detail()` | 无 | `None` | 全选/取消全选/反选右栏详情表的复选框列 |
| `_backup()` | 无 | `None` | 获取选中配置 → 收集文件 → 前置检查路径是否存在 → 创建 `BatchBackupWorker` 一次性处理所有配置 → 完成后弹出汇总对话框 |
| `_restore_selected()` | 无 | `None` | 遍历右栏详情表所有勾选项，弹出确认框后通过 QTimer 驱动逐次批量恢复 |
| `_restore_session()` | 无 | `None` | 遍历右栏详情表所有行（无视勾选），确认后通过 QTimer 驱动逐次恢复 |

### 私有方法流程

| 方法 | 核心逻辑 |
|------|----------|
| `_backup()` | 收集所有选中配置的文件字典；对路径全部不存在的配置弹窗告警（用户可取消）；创建 `BatchBackupWorker` 通过 QThreadPool 一次性后台处理所有配置 |
| `_batch_backup_done(storage, summary)` | 备份全部完成后启用按钮，调用 `_refresh_sessions(storage)`，弹出成功/跳过/失败的汇总对话框 |
| `_backup_error(msg)` | 弹出错误对话框，恢复按钮状态 |
| `_refresh_sessions(storage)` | 调用 `storage.list_sessions()` 获取按 session 分组的备份记录，填入左栏 `session_table`；在首行插入虚拟 session `__latest__`（各配置最新版本聚合） |
| `_on_session_selected()` | 左栏选中事件：从 `_session_data` 取出对应版本，填入右栏 `detail_table`，第 0 列为复选框（默认勾选） |
| `_select_all_detail()` / `_deselect_all_detail()` / `_invert_detail()` | 遍历右栏详情表所有行，批量设置复选框状态 |
| `_restore_selected()` | 检查 `_is_restoring` 锁；遍历右栏所有勾选项；确认后以 `(config_name, backup_id)` 填充 `_restore_session_queue`，通过 `QTimer.singleShot(0)` 驱动 `_restore_next_in_session` |
| `_restore_session()` | 检查 `_is_restoring` 锁；遍历右栏所有行（无视勾选）；同上 Timer 驱动恢复 |
| `_restore_next_in_session()` | 从队列首部弹出一个 `(config_name, backup_id)`，调用 `_restore_single` 启动 RestoreWorker；队列空时重置 `_is_restoring` 锁 |
| `_restore_single(config_name, backup_id)` | 检查 `_is_restoring` 锁 → 设锁 → 禁用按钮 → 创建 RestoreWorker + RestoreSignals → 启动 QThreadPool |
| `_restore_done(result)` | 启用按钮，清理 `_restore_signals`；如果 `_restore_session_queue` 非空则 `QTimer.singleShot(0, _restore_next_in_session)` 继续；否则解锁 `_is_restoring` |
| `_restore_error(msg)` / `_handle_blocked(msg, procs)` | 启用按钮，解锁 `_is_restoring`，清空队列 |
| `_get_selected_configs()` | 遍历勾选框返回已选中的 config 字典列表 |
| `_browse_target()` | 弹出目录选择对话框，更新目标路径标签 |

### 特别说明

- ⚠️ **QThreadPool + 信号生命周期**：`BatchBackupSignals` 和 `RestoreSignals` 均存储在 `self` 实例属性（`self._backup_signals` / `self._restore_signals`）上，防止跨线程信号因 Python GC 回收发送者 QObject 而被 Qt 丢弃。任务完成后在回调中主动置 `None` 允许 GC。
- ⚠️ **备份历史双栏**：左栏 `session_table` 显示备份批次（首行固定为虚拟 session「📌 最新配置」，聚合所有配置的最新版本）；右栏 `detail_table` 第 0 列为复选框，支持多选批量恢复。
- ⚠️ **批次恢复并发锁**：`_is_restoring` 标志位防重入，`_restore_selected` 和 `_restore_session` 入口均检查，避免同时执行多个恢复操作。
- ⚠️ **Timer 驱动而非信号链**：`_restore_done` 通过 `QTimer.singleShot(0, self._restore_next_in_session)` 触发下一项恢复，将回调与 RestoreWorker 信号解耦。
- 备份目标支持「本地目录」和「ZIP 打包」两种，通过 `target_combo` 切换。
- `target_path_label` 默认显示 `storage.base_dir`，可手动浏览修改。

## config_tab.py

文件职责：JSONC 配置规则编辑器。左侧 QListWidget 列出所有规则文件，右侧 QPlainTextEdit 编辑源码，实时校验语法。支持新建和删除规则。

### 公开接口

| 类/方法 | 参数 | 返回值 | 核心作用 |
|---------|------|--------|----------|
| `ConfigTab.__init__` | `config_dir: Path` | — | 初始化编辑器，调用 `refresh_rules` 加载规则列表 |
| `refresh_rules()` | 无 | `None` | 遍历 `builtin/` 和 `user/` 子目录下的 `.jsonc` 文件，填入左侧列表 |
| `_save()` | 无 | `None` | 校验当前编辑器内容语法，通过后写回 `_current_file` |
| `_validate()` | 无 | `None` | 实时用 `json5.loads` 验证编辑内容，更新验证标签文字和颜色 |
| `_scan()` | 无 | `None` | 调用 `scan_installed`，弹出匹配结果对话框 |
| `_new_rule()` | 无 | `None` | 写入模板到 `config/user/new_rule_N.jsonc`，自动选中 |
| `_delete_rule()` | 无 | `None` | 弹出确认对话框，确认后 `unlink()` 删除当前规则文件 |
| `_on_list_selected(current, previous)` | `current, previous: QListWidgetItem` | `None` | 左侧列表选中事件：加载对应文件内容到编辑器，设置 `_current_file` |

### 特别说明

- 界面为水平双栏：左侧 QListWidget 显示规则名（itemData 存文件路径），右侧编辑器 + 底部按钮行。
- 语法正确时验证标签显示绿色 `语法正确 ✔`；错误时显示红色 `语法错误: {异常信息}`。
- 新建规则固定写入 `user/` 子目录，内置规则（`builtin/` 下）可编辑、可删除。

## settings_tab.py

文件职责：设置面板。包含定时备份（注册/卸载 Windows 计划任务）、版本策略配置、恢复设置和关于信息。

### 公开接口

| 类/方法 | 参数 | 返回值 | 核心作用 |
|---------|------|--------|----------|
| `SettingsTab.__init__` | `storage: StorageBackend` | — | 初始化 UI 组件 |
| `_apply_scheduler()` | 无 | `None` | 开关开 → 调用 `_register_task`；关 → `_unregister_task` |
| `_register_task()` | 无 | `None` | 调用 `schtasks.exe /Create` 注册每日执行计划任务 |
| `_unregister_task()` | 无 | `None` | 调用 `schtasks.exe /Delete` 删除 `WinConfigBKP_DailyBackup` 任务 |
| `_reboot_replace()` | 无 | `None` | 弹出信息提示对话框 |

### 特别说明

- 定时任务名称硬编码为 `WinConfigBKP_DailyBackup`。
- `_register_task` 中脚本路径使用 `Path(__file__).resolve().parent.parent.parent / "main.py"`，打包后需调整为 exe 路径。
- `_unregister_task` 静默忽略 `schtasks.exe` 报错（任务不存在时不会弹出错误）。
