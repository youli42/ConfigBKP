# 配置文件 v1.2 — 多平台路径路由

## 目标

`paths` / `data_paths` 字段支持按平台分配不同路径，同一规则文件在不同系统上自动使用对应路径。

## 配置格式

```jsonc
{
  "name": "VS Code",
  "paths": {
    "windows": ["%APPDATA%\\Code\\User\\settings.json"],
    "macos": ["~/Library/Application Support/Code/User/settings.json"],
    "linux": ["~/.config/Code/User/settings.json"]
  },
  "data_paths": {
    "windows": [],
    "macos": [],
    "linux": []
  }
}
```

**兼容旧格式**：`paths` 仍可以是数组，代码自动识别类型并路由。

## 核心接口

```python
# config_parser.py
def resolve_path_for_platform(cfg: dict, key: str, system: str) -> list[str]:
    """从配置中取出 key（如 \"paths\"）的值，若为 dict 则按当前平台取值，否则原样返回"""
    val = cfg.get(key, [])
    if isinstance(val, dict):
        return val.get(system, val.get("default", []))
    return val
```

系统名取值：`platform.system().lower()` → `"windows"` / `"linux"` / `"darwin"`

### Step 1：核心解析函数

- [ ] `config_parser.py` 新增 `resolve_path_for_platform(cfg, key, system) → list[str]`

```python
def resolve_path_for_platform(cfg: dict, key: str, system: str = "") -> list[str]:
    if not system:
        system = platform.system().lower()
    val = cfg.get(key, [])
    if isinstance(val, list):
        return val
    if isinstance(val, dict):
        return val.get(system, val.get("default", []))
    return []
```
