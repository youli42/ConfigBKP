# 配置文件 v1.1

## 目标

扩展配置文件格式，增强 GUI 展示与交互，完善已有字段的实现。

## 新增字段

```jsonc
{
  // === 已有字段 ===
  "name": "VS Code",
  "description": "Visual Studio Code 编辑器配置",
  "version": 1,
  "enabled": true,

  // === 新增字段 ===
  "allow_running_backup": false,          // 是否允许程序运行时备份
  "data_paths": [],                       // 程序数据目录（区别于配置文件），默认空
  "backup_scope": {                       // 备份范围开关
    "config": true,                       //   true 时收集 paths
    "data": false                         //   true 时收集 data_paths
  },

  // === 已有，待实现 ===
  "strategy": {
    "type": "incremental",
    "max_versions": 10,
    "ignore_patterns": ["*.log", "*.tmp"]
  }
}
```

## 数据范围定义

| 字段 | 含义 | 收集时机 | 恢复目标 |
|------|------|----------|----------|
| `backup_scope.config` | 是否备份配置文件 | `true` → 遍历 `paths` | `config["paths"]` 中匹配的源路径 |
| `backup_scope.data` | 是否备份程序数据 | `true` → 遍历 `data_paths` | `config["data_paths"]` 中匹配的源路径 |

**收集规则**：
1. `backup_scope.config=true` → 收集 `paths` 中所有文件
2. `backup_scope.data=true` → 收集 `data_paths` 中所有文件
3. 同一文件被两列表都命中 → 去重（后遍历者覆盖前者的路径记录）
4. 两列表各自独立递归展开目录

**恢复规则**：
- metadata 中新增 `source_type: "config" | "data"` 标志，标记每条文件来自哪个列表
- 恢复时根据 `source_type` 选择对应的配置路径作为目标写入位置
- `restore_engine.py` 目标路径匹配逻辑需支持两个列表

  ```python
  files: dict[str, Path] = {}
  file_sources: dict[str, str] = {}       # rel_path → "config" | "data"

  if cfg.get("backup_scope", {}).get("config", True):
      for p in cfg.get("paths", []):
          for rel, abspath in _collect_files(p):
              files[rel] = abspath
              file_sources[rel] = "config"

  if cfg.get("backup_scope", {}).get("data", False):
      for p in cfg.get("data_paths", []):
          for rel, abspath in _collect_files(p):
              files[rel] = abspath
              file_sources.setdefault(rel, "data")  # 不覆盖 config 已有记录

  # metadata 写入时：
  # metadata["source_type"] = file_sources.get(rel_path, "config")
  ```

## 内置规则默认值

所有内置规则的 `data_paths` 初始为 `[]`，`backup_scope.data` 初始为 `false`，`allow_running_backup` 初始为 `false`。用户可在向导中按需开启。
